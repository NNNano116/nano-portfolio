## 5. 물리 모델 (구체, 프레임 루프)

각 구체는 `pos`/`vel`(THREE.Vector3)을 가지며, 매 프레임 힘을 적분한다.

### 5-1. 상시 힘 — 세 힘의 교차(crossfade)
`wMagnet/wFloat/wFlow = 0.7 + 0.3·sin(t·{0.12,0.1,0.08} + φ)` 로 강약 교차.

| 힘 | 식(요지) | 효과 |
|----|----------|------|
| **자석(응집)** | `acc += -2.85·pos·wMagnet` | 중심 스프링 |
| **부유** | `acc += sin(t·…+위상)·{4.4,4.4,2.6}·wFloat` | 위상 다른 잔잔한 떨림 |
| **흐름** | `vel += (flowAxis × pos)·0.55·wFlow·dt` | 축 둘레 소용돌이(약하게=발산 방지) |
| **감쇠 / 경계** | `vel*=0.18^dt` / `|pos|>7.6`면 되돌리고 `vel*0.82` | 프레임레이트 독립·이탈 방지 |

### 5-2. 상호작용 힘
| 입력 | 동작 |
|------|------|
| **커서 반발(상시)** | 반경 `r+4.2`(드래그 +1.6) 안 `f²·120·dt`로 밀어냄 |
| **드래그 휩쓸기** | 커서 화면속도 평활화 → 진행 방향으로 끌고 감(드래그 ×1·호버 ×0.45) |
| **클릭 버스트** | `pointerdown` 시 반경 `r+6` 안 `(1-d/R)·23` 방사 임펄스 |

### 5-3. 구체 간 상호작용 (O(n²))
- **분리**: 겹치면(`dist < a.er + c.er + gap`) 위치를 절반씩 밀어냄. **충돌 반지름은 `er`(현재 스케일 반지름)** — 인트로 중 작을 땐 안 부딪힘(§7).
- **표면장력**: 약간 떨어진(`< min+2.4`) 이웃을 `cohK=5.2·mNorm`로 끌어당김(모일 때만).
- **동적 간격**: `gap = 0.46 − 0.27·mNorm`. `mNorm = clamp((wMagnet−0.4)/0.6, 0, 1)`.

## 6. 속도 임계값 기반 반발 (구체 떨림 제거의 핵심) ⭐

가운데 뭉칠 때 **떨림(버즈)** 제거 위해 반발을 **접근속도에 연동**:
- `approach ≤ vLow(1.6)` → **e=0**(완전 흡수→정지, 떨림 0) / `vLow~vHigh(4.5)` 선형 → `≥vHigh` → e=`eMax(0.9)`(탱탱).
- 임펄스 `jimp=(1+e)·approach·0.5`. **e=0이면 상대 정상속도를 0으로 흡수**.

> ⚠️ 반발계수를 흡수 부족으로 두면 위치보정과 싸우며 떨림. `e=0`=완전 흡수 유지.

## 7. 인트로 — 구체 우아한 생성(제자리 + 약한 펼침) ⭐

구체는 날아오지 않고 **각자 자리에서 우아하게 생성**된다(겹쳐 통과·버벅임 없음).

- **스케일 인**: scale `0 → r`, `easeOutCubic`(끝속도 0=점프 없음), `introDur 0.7`. 불투명(반투명 페이드 없음).
- **가운데→바깥 리플**: `born = min(pos.length()/5, 1)·0.5` — 중심 구체부터 피어나 바깥으로.
- **버벅임 방지(핵심)**: 충돌을 **현재 스케일 반지름 `er = r·easeOutCubic(p)`** 로 판정(§5-3). 작을 땐 안 부딪혀, 자라는 만큼만 부드럽게 자리 양보 → 갑작스런 밀침/튐 없음.
- **약한 펼침**: 시작 위치 `pos×0.9`(중앙 10% 수축) + 등장 시 **바깥 방향 속도 `pos×0.8`**(거리 비례) → 자라며 살짝 펼쳐졌다 자석·감쇠로 균형점에 안착. (밋밋함 보완)
- **순서**: `t < born`이면 물리 정지(제자리 대기) → 등장 후 기존 물리(§5·§6) 인계.

## 9. 배경 레이저 (mattwilldev.com 참조) ⭐

대각 레이저가 **우상단에서 좌하향**으로 깔림. Pts.js 원본을 Playwright로 **픽셀 측정**해 재현. 별도 2D 캔버스·자체 rAF.

### 9-1. 핵심 사양 (실측 기반)
| 항목 | 값 | 근거 |
|------|-----|------|
| **각도** | **고정 63.1°**(`LASER_DEG`) | cross-correlation 측정 — 뷰포트 무관 고정 |
| **개수** | `min(W·0.07, 150)`(하한 40) | 1366px → ~96 |
| **상단 시작** | 항상 위쪽 가장자리(y=0), x 우측 치우침 `pow(rand,0.6)` | 상단행 우편중·edge 접촉 |
| **길이** | `(0.55–1.6)·H` 제각각 | 아래로 갈수록 성겨짐 |

### 9-2. 모션 / 색 / 인터랙션 / 모바일
- **드리프트**: 좌측으로 `13px/s`. 좌측 끝 `-0.08W` 넘으면 `+driftRange`로 재진입(새 길이).
- **우측 한계(모바일 커버리지)**: `driftRange = W·1.08 + H·0.55` — 레이저가 좌하향이라 길이(∝H)만큼 좌로 쓸리므로, **높이를 반영**해야 세로 긴 모바일에서도 **우측 하단까지 도달**(topX_max ≳ W + cot(63.1°)·H). ⚠️ 폭(W)만 쓰면 모바일 우하단 빈다.
- **선/점**: 선 흰색 `rgba(255,255,255,bright)` 두께1. 헤드 점(하단 끝) 핑크 `#FF3F8E`·시안 `#04C2C9`·블루 `#2E55C1`, r1.2, 글로우 없음.
- **커서 밝힘**: 커서–선 거리 `<40px` → `bright 0.1→0.25`(`+0.015`/프레임, 멀면 `−0.01`).
- **reduced-motion**: 드리프트 0.

## 10. 드래그 안내 인디케이터 (입력환경별 + 반응형) ⭐

하단 가운데. **마우스(흰선 캡슐) + 네온 라인 + 화살표**가 순차로 그려졌다 지워지는 안내(DOM/SVG `.main1__scroll`).
색·구조는 mattwilldev Lottie 추출, 시퀀스는 사용자 지정.

### 10-1. 구성 (SVG viewBox 44×66, 캡슐 y 1.5~64.5)
| 요소 | 클래스 | 색 | 동작 |
|------|--------|-----|------|
| 마우스 외곽선 | `.main1__m-mouse` | 흰선 `rgba(236,240,248,.85)` | **시계방향** draw/erase |
| 네온 라인 | `.main1__m-line` | 시안 `#00aeff`+글로우 | **위→아래** draw/erase |
| 화살표 머리 | `.main1__m-arrow` | 시안 `#00aeff`+글로우 | 등장 후 아래로 슬라이드+페이드 |

- draw=`dashoffset 100→0`, erase=`0→-100`(같은 방향), `pathLength=100`.
- 화살표 상하 대칭: 라인 `y16→44`, 팁 `y50`(상·하 여백 14.5 동일).

### 10-2. 시퀀스 (루프 4s)
마우스 시계방향 그리기(0~0.44s) → 네온 라인 위→아래(0.52~0.96s) → 화살표 등장(0.92~1.16s) → 화살표↓ 소멸(2.2~2.56s) → 라인 위→아래 소멸(2.48~2.84s) → 마우스 시계방향 소멸(2.88~3.28s) → **0.72s 정지** → 반복. 첫 등장은 `1.2s` 지연(인트로 후).

### 10-3. 입력환경별 라벨 + 반응형 크기
- **라벨(입력 상황별)**: PC(마우스)=`CLICK · SCROLL` / 터치(`@media (hover:none),(pointer:coarse)`)=`TOUCH & DRAG`. 두 span 토글.
- **반응형 크기(가로폭 비례 축소)**: SVG `width clamp(26px,7vw,44px)`(`aspect-ratio 44/66`), 라벨 `clamp(8px,2.3vw,11px)`, `gap clamp(6px,1.8vw,12px)`, `bottom clamp(18px,4vw,32px)`. → 데스크톱 44/11px, 모바일(390px) ≈ 27/9px.
- **reduced-motion**: 애니메이션 없이 완성형 정적.

| **패럴럭스** | 터치 기기(`pointer:coarse`)에선 JS에서 바로 종료(터치 후 텍스트 어긋남 방지) |
| **드래그 안내 크기** | SVG·라벨·간격·여백 `clamp(vw)` 축소(§10-3) |
| **드래그 안내 문구** | 입력환경별 CLICK·SCROLL / TOUCH & DRAG(§10-3) |
| **푸터** | 제거(미표시) |

## 12. 안정성 / 검증 (Playwright 직접 관찰·픽셀 측정)

- **구체**: spread 2.91–3.65 유지(발산 없음) · 차분 국면 평균속도 ~0.72(<vLow → 떨림 0) · 클릭 ~22/드래그 ~14로 반발 복구.
- **레이저 각도**: 자체 빌드 cross-correlation **63.1°**(사이트 동일), 개수 96.
- **레이저 모바일 커버리지**: 390×844 4분면 lit 픽셀 TL 3769 / TR 3472 / BL 2551 / **BR 3865**(이전 0) — 우하단 정상.
- **드래그 안내**: 화살표 bbox `y16→50`(여백 대칭) · 반응형 1366px 44/66·11px, 390px 27.3/40.9·9px.
- **반응형**: 서브타이틀 데스크톱 좌우여백 동일(중앙), 모바일 −10.5px. 푸터 DOM 없음.

> ⚠️ 검증 주의: 기존 dev 서버가 **stale(HMR 미반영)** 이면 수정이 화면에 안 보일 수 있음 → **클린 포트로 재기동**해 확인.
